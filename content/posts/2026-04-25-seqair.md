I've been working on [Rastair],
a bioinformatics tool for variant and methylation calling,
for a while now.
I previously wrote about it [here][rastair-post]
and about its GPU-based Random Forest inference [here][ml-post].
One of the big aspects of this is reading and writing files
in efficient and correct ways.

[Rastair]: https://www.rastair.com/ "Rastair website"
[rastair-post]: https://deterministic.space/rastair.html "Notes on Rastair, a variant and methylation caller"
[ml-post]: https://deterministic.space/gpu-random-forest-ml.html "Random Forest ML on GPU"

My focus was to make the developer experience very Rust-like.
I wanted to have strong types for everything
that enforce logic constraints,
distinguish "same" types with parameters,
and make sure that APIs can only be used in the right order
(using session types/type-state builders[^1]).

[^1]: This is one of my favorite features in Rust that I don't get to use so often.
  [I wrote about it][elegant-apis-in-rust] all the way back in 2016!

[elegant-apis-in-rust]: https://deterministic.space/elegant-apis-in-rust.html#session-types
